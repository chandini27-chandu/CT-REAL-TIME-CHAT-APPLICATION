## Description:

The Real-Time Chat Application is a modern web-based communication platform that enables users to exchange messages instantly using WebSocket technology. Unlike traditional web applications that rely on repeated HTTP requests to fetch new data, this application establishes a persistent WebSocket connection between the client and the server, allowing messages to be delivered immediately without refreshing the page. The primary objective of this project is to demonstrate real-time communication, responsive user interface design, and efficient client-server interaction using modern web development technologies.

The application is developed using React.js for the frontend and Node.js with the ws (WebSocket) library for the backend. React.js is selected because it provides a component-based architecture, making the application modular, reusable, and easy to maintain. Components such as the chat header, message list, individual message item, and message input are organized separately, improving readability and scalability. React's state management and lifecycle features are used to dynamically update the user interface whenever a new message is received.

For the backend, Node.js is chosen because of its event-driven and non-blocking architecture, which is highly suitable for handling multiple simultaneous WebSocket connections. The ws library is used to create the WebSocket server that manages client connections, broadcasts messages to all connected users, and maintains message history while the server is running. Whenever a client sends a message, the server receives it, stores it temporarily in memory, and broadcasts it to every connected client in real time.

The user interface is designed with a clean, modern, and responsive layout using CSS3. The application includes a professional chat interface with a header, message display section, and message input area. Messages are displayed in chat bubbles along with timestamps, providing a familiar messaging experience. The responsive design ensures that the application functions smoothly on desktops, tablets, and mobile devices by adapting the layout to different screen sizes.

The project follows a modular folder structure that separates frontend and backend code into independent directories. This organization improves maintainability and makes future enhancements easier. Since the project requirements focus on frontend development and WebSocket communication, no additional technologies such as databases or authentication systems are included. Message history is maintained temporarily in server memory during runtime, fulfilling the project requirement without introducing unnecessary complexity.
